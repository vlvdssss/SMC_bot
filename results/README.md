## Формат результатов

Каждый файл результатов (CSV) содержит:
- **Дата/время**: Когда был сигнал
- **Инструмент**: XAUUSD, US30 и т.д.
- **Сигнал**: BUY или SELL
- **Entry**: Цена входа
- **SL**: Stop Loss
- **TP**: Take Profit
- **Результат**: WIN/LOSS
- **PnL**: Прибыль/убыток в R или $
- **Balance**: Баланс после сделки

Пример:
```csv
time,symbol,signal,entry,sl,tp,result,pnl_r,balance
2023-01-05 09:00:00,XAUUSD,SELL,1850.50,1853.50,1844.50,WIN,2.0,1020.00
2023-01-06 14:00:00,XAUUSD,BUY,1845.20,1842.20,1851.20,LOSS,-1.0,1010.00
```

## Нейминг файлов

Используйте понятные имена:
- `q1_2023.csv` — квартал 1 (январь-март) 2023
- `full_2023.csv` — весь 2023 год
- `jan_2023.csv` — январь 2023
- `experiment_v2_2023.csv` — результаты эксперимента

## Анализ результатов

Для анализа используйте:
1. **Excel/Google Sheets** — базовый анализ (WR, DD, PnL)
2. **Python notebooks** — расширенный анализ (equity curve, распределение сделок)
3. **Документация** — описание в `docs/06_results.md`

## Метрики для отчетов

Обязательные метрики:
- **Total Trades**: Общее количество сделок
- **Win Rate**: % выигрышных сделок
- **Avg Win / Avg Loss**: Средний выигрыш/проигрыш
- **Profit Factor**: Отношение прибыли к убыткам
- **Max Drawdown**: Максимальная просадка (%)
- **Expectancy (R)**: Математическое ожидание в R

## Правила

### ✅ Что хранить:
- Результаты бэктестов
- Equity curves
- Статистика по сделкам
- Отчеты (если в CSV)

### ❌ Что НЕ хранить:
- Код стратегий (они в strategies/)
- Логи выполнения (они в logging/)
- Исторические данные (они в data/ или отдельно)
- Эксперименты (описания в docs/05_experiments.md)

## Сравнение годов

Для сравнения результатов по годам:
```
results/xauusd/2023/full_2023.csv → WR 65%, DD -8.3%
results/xauusd/2024/full_2024.csv → WR 62%, DD -7.5%
results/xauusd/2025/full_2025.csv → WR 68%, DD -6.1%
```

Это позволяет отслеживать стабильность стратегии во времени.
